## Контекст

Сейчас Sidebar показывает:
- Раздел "Книга" со всеми книгами (включая те, что в сериях)
- Раздел "Серии" с каждой серией и вложенными в неё книгами

Это создаёт дублирование: одна книга видна одновременно в двух местах.

Sprint 33 должна консолидировать эти представления в единую иерархию:
- Раздел "Книга" (заголовок)
  - Подраздел "Без серии" (книги без seriesId)
  - Серия 1 (с вложенными книгами)
  - Серия 2 (с вложенными книгами)
  - ...

Это соответствует ментальной модели пользователя: книги сгруппированы по сериям.

## Scope

### Allowed paths (ТОЛЬКО):

- apps/studio/src/components/Sidebar.tsx (полная переработка структуры)
- apps/studio/src/app/page.tsx (передать данные в Sidebar по-новому)

### Forbidden paths:

- apps/studio/prisma/** (Book.seriesId уже существует)
- Новые компоненты

## Objective

Переорганизовать структуру Sidebar для единой иерархии Series/Books:

1. **Новая структура Sidebar:**
   ```
   Книга
     Без серии
       [book 1 without series]
       [book 2 without series]
     Серия "Мистика"
       [book A in series]
       [book B in series]
     Серия "Приключения"
       [book C in series]
   ```

2. **Раздел "Без серии":**
   - Заголовок "Без серии" (как подраздел, стиль светлее чем Серия)
   - Список книг WHERE seriesId IS NULL
   - Может быть свёрнут/развёрнут (collapse toggle)
   - Если нет книг — не показывается

3. **Серии (как раньше в "Серии" разделе):**
   - Заголовок Серии (название)
   - Кнопка collapse/expand (▾/▸)
   - Вложенные книги этой серии
   - Кнопка редактирования серии (gear или inline)

4. **Удалить раздел "Серии":**
   - Старый отдельный раздел "Серии" больше не показывается
   - Вся логика серий переходит в раздел "Книга"

5. **Кнопки действий:**
   - Рядом с "Без серии" — кнопка создания новой книги (если её там нет)
   - Рядом с "Книга" — кнопка создания новой серии (оставить как есть)
   - Рядом с каждой серией — кнопка редактирования/удаления серии

## Rules

1. Books отображаются ТОЛЬКО один раз (либо в "Без серии", либо в серии)
2. Порядок: сначала "Без серии", потом серии в порядке Series.order
3. Collapse state для "Без серии" и для каждой серии отдельно (lift in page.tsx)
4. Не трогать логику drag-drop (если была)
5. Не трогать логику создания/редактирования серии

## Validation

Живая проверка в браузере:

1. **Загрузка:**
   - Раздел "Книга" показывает иерархию Без серии → Серии
   - Раздел "Серии" НЕ показывается (удалён)

2. **Без серии:**
   - Книги без seriesId видны в подразделе "Без серии"
   - Клик на книгу → выбирается как activeBook

3. **Серии:**
   - Каждая серия показана как заголовок (развёрнуто/свёрнуто)
   - Клик на ▾/▸ → toggle collapse
   - Книги под серией видны/скрыты правильно

4. **Управление:**
   - Кнопка + создания новой книги работает
   - Кнопка редактирования серии работает
   - Перетаскивание книги между сериями (если было) работает

5. **Persist:**
   - После refresh — collapse state сохранён для каждой серии
   - activeBook ID сохранён
