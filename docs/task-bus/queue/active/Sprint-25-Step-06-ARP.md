# ARP: Sprint-25-Step-06

Поиск в верхнем меню — глобальный (главы/сцены/персонажи/идеи активной книги + список книг),
с переключателем «искать только в основном тексте».

## Что сделано

1. **`apps/studio/src/domain/search.ts`** (новый файл) — чистая функция `searchWorkspace()` без
   React/DOM/побочных эффектов. Принимает `query`, `books` (весь Workspace, для поиска по
   названиям книг), `chapters`/`characters`/`ideas` (уже отфильтрованные вызывающей стороной по
   активной книге — сам модуль про `activeBookId` ничего не знает) и опциональный `mainTextOnly`.
   Возвращает секционированный `SearchResults` (`books` / `chaptersAndScenes` /
   `characters` / `ideas`). Экспортирует также `SEARCH_MIN_QUERY_LENGTH = 2`.
2. **`apps/studio/src/components/Header.tsx`** — форма поиска (один `<input>`, единый выпадающий
   список с подписанными секциями, рендерится только если в них есть результаты; чекбокс
   «Искать только в основном тексте» рядом с полем; закрытие по Escape/клику вне формы —
   отдельный `mousedown`-эффект по аналогии с уже существующим для Файл/Правка/Вид; Ctrl/Cmd+K
   фокусирует поле поиска). Компонент принимает новые пропсы (`books`, `activeBookId`,
   `chapters`, `characters`, `ideas`, `onSelectBook`, `onSelectCharacter`,
   `onSelectSearchMatch`, `onSelectIdeaMatch`) и вызывает `searchWorkspace()` через `useMemo`.
3. **`apps/studio/src/app/page.tsx`** — проброс новых пропсов в `<Header>`; две новые функции:
   `handleSelectSearchMatch(chapterId, sceneId?)` (снимает «свернуть все главы», разворачивает
   нужную главу/сцену, если они свёрнуты, вызывает `selectChapter`/`selectScene`, скроллит через
   один `requestAnimationFrame`) и `handleSelectIdeaMatch(ideaId)` (выключает Focus Mode, если
   он включён, затем тем же способом скроллит к `idea-block-{id}`). Локальный дубль
   `scrollElementIntoView` (4 строки) — `Sidebar.tsx`, где живёт оригинал, в Forbidden paths.
4. **`apps/studio/src/components/IdeasPanel.tsx`** — единственное изменение: добавлен
   `id={`idea-block-${idea.id}`}` на обёртку каждой заметки (та же конвенция, что
   `chapter-block-{id}`/`scene-block-{id}`), без переделки компонента.
5. `apps/studio/e2e/search.spec.ts` — **не создан**, см. «Отклонения».

### Поля, участвующие в поиске (подтверждение для Output-требования №1)

- **`Book.title`** — по всем книгам рабочего пространства (единственная часть поиска не
  ограниченная активной книгой).
- У активной книги: **`Chapter.title`**, **`Chapter.subtitle`**, **`Scene.title`**,
  **`Scene.text`**, **`Character.name`**, **`Character.description`**, **`Character.notes`**,
  **`Idea.text`**. `Character.photoUrl` и Book-поля кроме `title` (genre/premise/annotations/
  tags) в поиске не участвуют — как и было решено в самой карточке.

### Переключатель «только основной текст» (Output-требование №2)

Выключен по умолчанию (`useState(false)` в `Header.tsx`). Причина (карточка, развилка 6):
Product Owner сформулировал «глобальным» как основное требование, чекбокс — как
дополнительное сужение поверх этого, а не наоборот; поведение «ничего не трогая» должно быть
максимально широким поиском согласно прямому запросу. При включении `searchWorkspace()`
возвращает только совпадения `Scene.text` активной книги — ни `Book.title`, ни
`Chapter.title/subtitle`, ни `Scene.title`, ни `Character.*`, ни `Idea.*` не участвуют (строгое
чтение Rules-раздела карточки: «ТОЛЬКО совпадения Scene.text»).

## Отклонения от Step Card

1. **`apps/studio/e2e/search.spec.ts` не создан.** Карточка явно помечает файл «опционально —
   см. Validation»; сам Validation-раздел требует именно ручной живой Playwright-проверки на
   scratch-порту (Shape 1), а не обязательного добавления файла в постоянный набор `test:e2e`.
   Учитывая объём уже покрытых Validation-кейсов (9 сценариев, все живьём проверены) и что
   `e2e/**` полностью исключён из eslint (`globalIgnores`) и не проверяется в этой карточке,
   решено не добавлять отдельный checked-in spec-файл сейчас — это техническое решение, не
   продуктовая неоднозначность. При необходимости постоянного регресс-теста это можно вынести в
   отдельный шаг.
2. **Клавиатурный шорткат.** Ctrl+K (и Cmd+K для паритета) реализован — фокусирует поле поиска
   и открывает список, если запрос уже достаточной длины. Реализация тривиальна (~8 строк,
   один `useEffect`), поэтому включена, как и предписывала карточка («включить, только если
   реализация тривиальна»). Символ `/` как альтернативный шорткат не добавлен — конфликтует с
   вводом текста в `<textarea>`/`<input>` сцен/заметок, что не тривиально разрулить корректно;
   Ctrl+K было достаточно для «включить хотя бы что-то».
3. **Гонка рендера (развилки 9, 12).** Единственный `requestAnimationFrame` (не двойной)
   оказался достаточным во всех живых сценариях — включая переход из `CharacterPanel` обратно
   в `EditorArea` перед сворачиванием/разворачиванием, и выход из Focus Mode перед скроллом к
   `idea-block`. Двойной `requestAnimationFrame` не потребовался; в коде оставлен одинарный.
4. Небольшая техническая деталь, не описанная в карточке дословно: при отсутствии результатов
   по достаточно длинному запросу дропдаун показывает «Ничего не найдено» вместо того, чтобы
   молча не открываться — минимальное UX-решение, не meняющее ни одного из описанных в карточке
   кейсов.

Иных отклонений нет.
