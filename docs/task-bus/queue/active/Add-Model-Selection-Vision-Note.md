## Objective

Добавить новый раздел дословно:

```markdown
## 14. Выбор ИИ-провайдера и модели (идея, не спроектировано)

Product Owner: нужна возможность выбора разных ИИ и моделей — не
только разные модели Claude, а разные провайдеры вообще (включая
DeepSeek). Нужен набор платных и бесплатных вариантов, плюс
"коннектор" для добавления новых провайдеров позже.

**Предложенный стартовый набор (актуально на 2026-07-06, проверено
поиском — цены и модели меняются быстро, не считать окончательным):**

- **Платные (топ-качество):** Anthropic Claude (уже используется) —
  Opus/Sonnet/Haiku; OpenAI GPT-5.5; Google Gemini 3.1 Pro.
- **Бюджетные (дёшево, не бесплатно по API):** DeepSeek V4 Flash —
  среди самых дешёвых вариантов с достойным качеством.
- **Бесплатные:** Google Gemini (бесплатный тариф с ограничением
  частоты запросов); OpenRouter (агрегатор — под одним API доступны
  по-настоящему бесплатные модели: DeepSeek R1, Llama 3.3 70B,
  Gemma 3 и другие).

**Архитектурная идея — коннектор:** единый интерфейс-адаптер (грубо:
"отправить промпт + системный промпт → получить текст"), под который
каждый провайдер реализует свой адаптер. Добавление нового провайдера
в будущем — это один новый адаптер, а не правка во всех route.ts
файлах Expert'ов сразу.

**Связь с уже зафиксированным:**
- Настройка выбора модели, вероятно, будет жить в пункте "Настройки"
  строки меню приложения (раздел 11, к Sprint 30) — не обязательно
  тот же срок, просто логическое место.
- Одна настройка на всех помощников сразу, или у каждого своя —
  вопрос пока не решён, всплывёт при проектировании.

**Статус:** идея и стартовый набор зафиксированы, ничего не
спроектировано и не приоритизировано. Не входит в текущий Sprint 12
(Co-author/Editor context) — отдельная будущая архитектурная работа.
```
