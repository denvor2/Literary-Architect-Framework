  ИСКЛЮЧИТЕЛЬНО на собственный `GET` ARP-сессии как источник истины о том, что "реальное", без

  операции над чужими продакшн-данными это недостаточно.
- Прямая проверка живого сервера (порт 3000, сейчас активно используется — установленные TCP-
  соединения) показывает 4 книги с содержательным реальным контентом: "0. Wild Mind", "1 Слон,
  которого не было", "2 Законы Тералии", "3 Проект Уллис" (ids "1".."4"). Это не совпадает с
  финальным утверждением ARP "БД после уборки содержит ровно исходные 'Книга 1'/'Вторая книга 2'"
  (2 книги). Важная деталь кода: id новой книги генерируется как `String(books.length + 1)`
  (`useWorkspaceController.ts`, не max/uuid) — то есть если на момент инцидента реально
  существовало 4 книги (ids 1-4), а ARP отфильтровал(а) "оставить только id 1 и 2", это тихо
  удалило бы 2 реальные книги; любая книга, созданная позже, заново получила бы id "3"/"4" по
  тому же счётчику длины массива — то есть ровно то расхождение, которое сейчас наблюдается,

- Расхождение может быть полностью безобидным (Product Owner мог переименовать/добавить книги
