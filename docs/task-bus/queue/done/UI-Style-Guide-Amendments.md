## Поправка к Sprint-10-Step-03 (если ещё не закоммичен)

Кнопка "Удалить персонажа" в CharacterPanel.tsx уже описана в
Step-02 с рамкой (border-zinc-300) — это правильно по признаку
"кнопка = рамка/заливка", но НЕ по цветовой семантике: разрушительное
действие должно быть красным акцентом, не нейтральным zinc. Поменяй
цвет рамки/текста на red-* (например border-red-300 text-red-600,
или аналогичный по духу остальной палитры оттенок) — сохрани форму
(rounded-full, padding) как есть, поменяй только цвет.

Если Step-03 уже закоммичен на момент, когда это прочитаешь — не
переоткрывай его, просто внеси это отдельным маленьким диффом и
опиши в новом ARP как самостоятельную правку.

## Поправка к Sprint-10-Step-04 и его Amendment (ещё не начаты)

В обоих файлах написано: кнопки "+ New Chapter" и "+ New Book" —
"тот же визуальный паттерн, что "+ New Character"". Это ссылка на
СТАРЫЙ стиль (текст-ссылка без рамки), который сам будет переделан
в Add-UI-Style-Guide.md. Правильная инструкция: обе новые кнопки
("+ New Chapter", "+ New Book") сразу делать контурными (border,
padding, rounded) — по конвенции Add-UI-Style-Guide.md, а не
копировать (уже устаревший к тому моменту) паттерн "+ New Character".

Если возьмёшь Add-UI-Style-Guide.md раньше Step-04 — тогда просто
скопируй уже переделанный класс кнопки "+ New Character" (он к тому
моменту будет правильным) для новых кнопок. Если порядок будет
обратный — сразу примени контурный стиль по этой поправке, не жди.
