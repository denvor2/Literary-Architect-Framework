## Scope

## Objective

Реальный баг, подтверждён скриншотом Product Owner: при выборе
"Co-author" в выпадающем списке кнопка всё равно показывает
"Редактор". Причина — текст кнопки жёстко зашит строкой "Редактор",
никогда не берётся из выбранного mode/info.

Найти:
```typescript
{status === "loading" ? info.preamble : "Редактор"}
```

Заменить на текст, отражающий реальный выбранный режим — например,
использовать сам mode напрямую (\`mode\`, уже доступен в этой области
видимости) вместо жёстко зашитой строки:
```typescript
{status === "loading" ? info.preamble : mode}
```

(Если по стилю лучше показывать что-то другое, чем сырое значение
mode — опиши альтернативу в ARP и обоснуй, но по умолчанию — просто
показать актуальный режим, а не всегда одно и то же слово.)

## Rules

- Минимальное изменение — только текст кнопки.
- Не трогай остальную логику/вёрстку.

## Validation

- npm run build / npm run lint / prettier --check — чисто.
- Живая проверка: выбрать "Co-author" в списке — кнопка показывает
  соответствующий текст, не "Редактор"; то же для Critic/Reader/Editor.
- Приложи изменённый фрагмент.

ARP файлом в docs/task-bus/queue/active/ + в чат.
