## Контекст

Sprint 33 Step 04 добавила иерархию Sidebar: раздел "Книга" со структурой:
- Без серии (подраздел)
  - [книги без seriesId]
- Серия 1
  - [книги с seriesId = Серия 1]
- Серия 2
  - [книги с seriesId = Серия 2]

Сейчас структура статична: книги показаны в их текущих местах (Без серии или в конкретной серии), 
но пользователь не может перемещать книгу из одной серии в другую или из "Без серии" в серию 
через интерфейс.

Sprint 33 Step 07 добавляет drag-drop функциональность:
- Пользователь может перетащить книгу на другую серию (или на "Без серии")
- При отпускании книга перемещается в целевую серию
- Состояние синхронизируется с БД (Book.seriesId обновляется)

## Scope

## Objective

Реализовать drag-drop для перемещения книг между сериями:

### 1. Drag-drop логика (HTML5):
   - Каждый элемент книги `<li>` получает `draggable="true"`
   - На книге: обработчик `onDragStart` → сохранить bookId в `dataTransfer.setData("bookId", ...)`
   - На целях (серия или "Без серии"): обработчик `onDragOver` → `e.preventDefault()` (разрешить drop)
   - На целях: обработчик `onDrop` → получить bookId, вызвать `onMoveBookToSeries(bookId, targetSeriesId || null)`
   - На целях: обработчик `onDragLeave` → отменить visual feedback

### 2. Visual feedback:
   - При drag: книга становится полупрозрачной (opacity: 0.5)
   - При hover над целью (серия/Без серии): целевой элемент подсвечивается (border, bg-color)
   - Drop-zone (серия или "Без серии") получает `border-dashed`, `bg-zinc-100/dark:bg-zinc-900` при drag-over

### 3. Callback в page.tsx:
   - Новый callback `onMoveBookToSeries(bookId: string, targetSeriesId: string | null)`
   - Вызывает `workspaceController.moveBookToSeries(bookId, targetSeriesId)` или обновляет `books` напрямую
   - После обновления: setState → Sidebar re-renders с новой структурой

### 4. Логика в useWorkspaceController:
   - Добавить метод `moveBookToSeries(bookId, seriesId)`:
     - Найти книгу в `workspace.books` по bookId
     - Обновить `book.seriesId = seriesId` (или null если "Без серии")
     - Сохранить через `workspaceStorage.saveWorkspace()`
     - Отправить PUT запрос на `/api/book-field` (или новый `/api/book-series`) с новым seriesId

### 5. API endpoint (опционально, если нужна):
   - PUT `/api/book-field` уже обновляет отдельные поля книги
   - Или создать `/api/book-series/{bookId}` → { seriesId: string | null }

### 6. Ограничения:
   - Нельзя перетащить книгу на саму себя (в своей же серии)
   - Нельзя перетащить на закрытую (свёрнутую) серию (раскрыть или отклонить drop)
   - После drop: немедленно обновить UI (не ждать server response)

## Rules

1. **Drag source:** только элемент книги `<li key={book.id}>` имеет `draggable="true"`
2. **Drop targets:** серия (заголовок или контейнер) и "Без серии" (контейнер)
3. **Visual feedback:** прозрачность при drag, подсветка при hover над целью
4. **Не закрытые целевые серии:** нельзя drop-ить на закрытую серию (она должна быть открыта)
5. **Сохранение:** немедленное обновление локального state, асинхронное сохранение в БД
6. **Откат:** если сохранение на сервер не удалось — откатить локальное изменение (или показать warning)
7. **Конфликты:** если seriesId на сервере уже изменён — принять server-side версию (last-write-wins)

## Validation

Живая проверка в браузере:

1. **Начало drag:**
   - Наведи курсор на книгу → drag-handle (можно на всю книгу)
   - Зажми мышь и начни перетаскивать → книга становится полупрозрачной

2. **Hover над серией:**
   - При перемещении над заголовком серии → граница становится пунктирной, фон подсвечивается
   - Курсор изменяется на "drop allowed"

3. **Перемещение между сериями:**
   - Перетащи книгу из Серии 1 → отпусти на Серию 2
   - Книга должна исчезнуть из Серии 1 и появиться в Серии 2 (немедленно в UI)
   - Refresh страницы → книга остаётся в Серии 2 (persisted в БД)

4. **Перемещение в "Без серии":**
   - Перетащи книгу из серии → отпусти на "Без серии"
   - Книга должна переместиться в подраздел "Без серии"
   - Refresh страницы → книга остаётся в "Без серии"

5. **Перемещение из "Без серии":**
   - Перетащи книгу из "Без серии" → отпусти на серию
   - Книга перемещается в цель серию

6. **Отмена drag:**
   - Если отпустить мышь в пустом пространстве (вне целей) → книга остаётся на месте

7. **Закрытая серия:**
   - Закрой серию (collapse)
   - Попытайся перетащить книгу на закрытую серию → drop не работает или открывает её

8. **Множественные книги:**
   - Создай несколько книг в разных сериях
   - Перемещай их между сериями → все операции работают независимо
