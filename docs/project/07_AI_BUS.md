# Глава 7. AI Bus

## Назначение

AI Bus — это протокол взаимодействия между Архитектором (ChatGPT), Claude Code и кодовой базой.

Его задача — сделать разработку:
- воспроизводимой;
- проверяемой;
- без повторной работы;
- без архитектурного дрейфа.

## Роли

### Архитектор (ChatGPT)

Отвечает за:

- архитектуру;
- декомпозицию;
- проектирование;
- ревью;
- принятие решений;
- планирование спринтов.

Не пишет большие объёмы production-кода.

### Claude Code

Отвечает за:

- реализацию;
- рефакторинг;
- тестирование;
- сборку;
- исправление ошибок.

Не принимает архитектурных решений самостоятельно.

## Цикл работы

1. Архитектор формирует Sprint.
2. Claude реализует один шаг.
3. Claude возвращает ARP-отчёт.
4. Архитектор принимает или отклоняет.
5. Только после этого начинается следующий шаг.

## Правила

- один шаг = один законченный результат;
- никаких скрытых изменений;
- никаких "заодно";
- каждый шаг проходит build/lint;
- каждый шаг проверяется архитектурно.

## Формат ARP

- STATUS
- SUMMARY
- FILES MODIFIED
- VALIDATION
- RISKS
- SYSTEM STATE
- NEXT STEP

Это обязательный контракт общения.

## Что запрещено

- менять архитектуру без согласования;
- выполнять следующий шаг без утверждения;
- скрывать риски;
- придумывать функциональность.

AI Bus является обязательной частью проекта.
