## Контекст

Step-01 реализовала мягкое удаление на уровне логики и репозитория. 

Этот step добавляет UI для корзины:
- Новый раздел "Корзина" в Sidebar (отдельный от активных книг)
- Кнопки восстановления (↩️) и окончательного удаления (🗑️) для каждой удалённой книги
- Счётчик удалённых книг

## Scope

### Allowed paths (ТОЛЬКО):

- apps/studio/src/app/page.tsx (передать deletedBooks и callbacks в Sidebar)

### Forbidden paths:

- apps/studio/prisma/** (schema уже done в Step-01)
- apps/studio/src/repositories/** (repo logic done в Step-01)
- Новые компоненты (если нужны, то inline в Sidebar, не отдельные файлы)

## Objective

Добавить UI раздела "Корзина" в Sidebar для отображения удалённых книг и действий над ними:

1. **Новый раздел в Sidebar** (после раздела "Серии"):
   - Заголовок "Корзина" + счётчик удалённых (если > 0)
   - Если нет удалённых книг — "Корзина пуста"
   - Если есть — список удалённых книг с иконками действий

2. **Для каждой удалённой книги:**
   - Название + дату удаления (deletedAt)
   - Кнопка восстановления (↩️): вызывает restoreBook(), обновляет UI
   - Кнопка окончательного удаления (🗑️): подтверждение + permanentlyDeleteBook()

3. **Стили:**
   - Удалённые книги выглядят более тусклыми (text-zinc-400, фон светлее)
   - Действия (кнопки) яркие/явные (hover effects как в активных книгах)

4. **Взаимодействие:**
   - Восстановление: книга возвращается в главный список, исчезает из Корзины
   - Окончательное удаление: confirm диалог ("Безвозвратно удалить?") → hard delete

## Rules

1. deletedBooks грузятся через useWorkspaceController (loadDeletedBooksForUser())
2. После action (restore/delete) — автоматическое обновление UI (state update в контроллере)
3. Нет персистентного collapse state для Корзины (всегда развёрнута при загрузке, как новый раздел)
4. Даты удаления показываются в локальном формате (DD.MM.YYYY или relative "2 дня назад")

## Validation

Живая проверка в браузере:

1. **Загрузка:**
   - Sidebar отображает раздел "Корзина" (пусто при первой загрузке)

2. **Удаление книги:**
   - Нажать на 🗑️ рядом с активной книгой (существующая кнопка)
   - Книга исчезает из главного списка
   - Появляется в Корзине с датой удаления

3. **Восстановление:**
   - В Корзине нажать ↩️
   - Книга возвращается в главный список
   - Исчезает из Корзины

4. **Окончательное удаление:**
   - В Корзине нажать 🗑️
   - Confirm диалог "Безвозвратно удалить…?"
   - После подтверждения → исчезает из Корзины

5. **Данные сохранены:**
   - После refresh — Корзина отражает правильное состояние
   - deleted_at поле корректно в БД (если доступ к psql)
