## Контекст

Форма глобального поиска (Sprint-25-Step-06) имеет чекбокс "искать только в основном тексте"
рядом с полем. Product Owner хочет переместить его под поле для лучшей читаемости и добавить
иконку-кнопку поиска.

## Scope

Allowed paths:
- apps/studio/src/components/Header.tsx (форма поиска)
- apps/studio/src/app/globals.css (если нужны новые стили)

Forbidden paths:
- Domain logic
- Search function (apps/studio/src/domain/search.ts)
- API endpoints

## Rules

- Чекбокс "искать только в основном тексте" переместить ПОД поле ввода (вместо рядом)
- Добавить иконку-кнопку поиска (Search icon из lucide-react) справа от поля
  - Размер: h-4 w-4 (или пропорциональный размер поля)
  - Цвет: zinc-500 (text-zinc-500 dark:text-zinc-400)
  - При клике: выполнить поиск (если не выполняется автоматически на каждый символ)
- Чекбокс сохраняет текущее поведение (toggle main-text-only фильтр)
- Layout: колонка (column) вместо строки (row)

## Validation

- npx tsc --noEmit - чисто
- npx eslint src - чисто
- npm run build - чисто
- Живая проверка (npm run dev):
  1. Открыть поиск (Ctrl/Cmd+K в Header)
  2. Проверить, что поле ввода сверху, чекбокс снизу
  3. Проверить иконку поиска справа от поля
  4. Ввести текст — поиск работает
  5. Кликнуть на иконку поиска — ничего не ломается (или выполняет поиск)
  6. Включить/выключить чекбокс — фильтр работает
  7. Проверить темный режим (иконка видна)
