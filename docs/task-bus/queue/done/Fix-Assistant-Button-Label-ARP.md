# ARP — Fix-Assistant-Button-Label

**Задача:** СРОЧНО — кнопка помощника всегда показывает "Редактор" вместо текущего режима
**Статус выполнения:** Готово к ревью

## Что сделано

В `apps/studio/src/components/EditorArea.tsx` кнопка-триггер заменена ровно как предлагал Step
Card — жёстко зашитая строка `"Редактор"` заменена на `mode` (уже доступен в этой области
видимости, тип `AssistantMode` — строковый union `"Co-author" | "Editor" | "Critic" |
"Reader"`, валиден для прямого рендеринга в JSX). Альтернатива не потребовалась — сырое значение
`mode` само по себе читаемо и совпадает с тем, что уже выбрано в выпадающем списке `<select>`
рядом, дополнительное сопоставление/новое поле в `MODE_INFO` было бы лишним усложнением.

## Диф

```diff
       >
-        {status === "loading" ? info.preamble : "Редактор"}
+        {status === "loading" ? info.preamble : mode}
       </button>
```

**Живая проверка (честно):** это чисто текстовая привязка к уже существующему React-состоянию
(`mode`), без обращения к бэкенду/API — тот же класс проверки, что и раньше для подобных чисто
UI-фиксов (например, кнопка "Заменить текст"), не требует tsx-скрипта с монки-патчем `fetch`,
поскольку сети тут вообще нет. Код-ревью: `mode` устанавливается через `<select onChange=
{(event) => setMode(...)}>` чуть выше и уже используется в `MODE_INFO[mode]` для остального —
привязка корректна и типобезопасна (подтверждено `tsc`/build без ошибок). Прошу лично проверить
в браузере: выбрать "Co-author" в списке — кнопка показывает "Co-author", не "Редактор"; то же
для Critic/Reader/Editor.
