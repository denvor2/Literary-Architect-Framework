## КРИТИЧЕСКИ ВАЖНО — прочитай перед началом

Термин "AI Bus" сейчас означает ДВЕ разные вещи в этом репозитории:

1. **ПРОЦЕСС разработки** (docs/ai-bus/, очередь Step Card/ARP/REVIEW
   между Architect и Programmer) — ПЕРЕИМЕНОВАТЬ в "Task Bus".
2. **Код продукта** (apps/studio/src/ai/aiBus.ts, реальный диспетчер
   AIOperation → Line Editor/Critic/Reader) — ОСТАЁТСЯ "AI Bus",
   НЕ ТРОГАТЬ.

НЕ делай глобальный find-replace. Каждое вхождение "AI Bus"/"ai-bus"
нужно вручную классифицировать по контексту.

## Scope

Allowed paths:
- docs/ai-bus/** (включая переименование самой папки → docs/task-bus/)
- docs/project/*.md (только там, где встречается ПРОЦЕССНЫЙ смысл)
- docs/adr/ADR-0004-*, ADR-0005-* (только если там есть процессные
  упоминания вперемешку с продуктовыми — скорее всего их там нет,
  эти ADR про продуктовый AI Bus, оставь как есть, если сомневаешься —
  не трогай и укажи в ARP)

Forbidden paths:
- apps/studio/** — весь код продукта, включая aiBus.ts. НЕ ТРОГАТЬ
  ни единой строки, даже если там встречается слово "AI Bus" в
  комментариях, описывающих продуктовый диспетчер.
- docs/product/DOMAIN_MODEL.md, docs/product/EXPERT_CATALOG.md — эти
  документы про продукт, "AI Bus" там означает код, не трогай.

## Objective

### Шаг 1 — инвентаризация (сделай и приложи в ARP ДО переименования)

grep -rn "AI Bus\|ai-bus\|AI_BUS" docs/ — приложи полный список
найденных строк с файлом и номером строки. Для каждой — одним словом
пометь: PROCESS (переименовать) или PRODUCT (оставить). Если
сомневаешься в конкретной строке — не переименовывай её, пометь как
UNCERTAIN и опиши сомнение в ARP, дождись моего решения.

### Шаг 2 — переименование (только PROCESS-вхождения)

- git mv docs/ai-bus/ docs/task-bus/
- git mv docs/task-bus/AI_BUS_V4.md docs/task-bus/TASK_BUS_V4.md
- Внутри docs/task-bus/**: "AI Bus" → "Task Bus", "AI_BUS" → "TASK_BUS"
  (только там, где это описывает процесс/очередь, не код).
- Обновить ссылки на переименованные пути в других файлах (например,
  если PROJECT_CHARTER.md или HANDOVER.md ссылаются на
  docs/ai-bus/... — поправить путь).
- docs/project/GLOSSARY.md: обновить запись про "AI Bus" (сейчас
  она про процесс) → "Task Bus", путь docs/task-bus/.

### Шаг 3 — не трогать

Любое упоминание, описывающее apps/studio/src/ai/aiBus.ts или его
поведение (диспетчеризацию AIOperation) — остаётся "AI Bus" везде,
включая ADR-0004, ADR-0005, если они там встретятся.

## Validation

- Полный инвентарный список из Шага 1 приложен в ARP ЦЕЛИКОМ, с
  пометками PROCESS/PRODUCT/UNCERTAIN по каждой строке.
- git status --short — только docs/**, apps/studio/** отсутствует.
- grep -rn "docs/ai-bus" . (кроме .git/) → 0 совпадений (все ссылки
  на старый путь обновлены).
- grep -c "AI Bus" apps/studio/src/ai/aiBus.ts до и после — число не
  изменилось (файл не тронут вообще, можно просто не включать его в
  diff).
- npm run build — чисто (переименование документации не должно
  ничего сломать, но проверь на всякий случай, вдруг где-то
  markdown-ссылка используется в коде).

## Output

ARP файлом в docs/ai-bus/queue/active/ (пока папка ещё называется
по-старому — сам переименовывающий Step Card, естественно, начинает
работу до переименования) + в чат.

## Stop Condition

Не коммить до STATUS: OK от Architect. Если по Шагу 1 найдутся
UNCERTAIN случаи — остановись и жди моего решения по ним, не
переименовывай наугад.
