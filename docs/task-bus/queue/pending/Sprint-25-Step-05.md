## Исполнитель этого шага - НЕ step-executor

Этот Step Card должен исполняться через subagent `ui-specialist` (.claude/agents/ui-specialist.md
+ skill literary-studio-ui-specialist), а не через обычный step-executor - оба добавлены в этой
же сессии специально для такого рода прохода. Тот, кто берёт эту карточку в работу, должен явно

почему сейчас) - см. правило в CLAUDE.md "Subagent does not see this conversation".

## Контекст и порядок исполнения

Product Owner (2026-07-11) хочет "привести дизайн в порядок, внедрить иконки, общий стиль" -
финальный проход по визуальному единообразию для всего apps/studio.

**Обязательное условие: этот шаг выполняется ПОСЛЕ того, как Sprint-25-Step-01 через
Sprint-25-Step-04 приземлятся (закоммичены), не раньше.** Полировка иконок/стиля на
компонентах, которые вот-вот структурно поменяются в этом же спринте (Sidebar получает новую
секцию, AssistantPanel меняет picker, Header получает строку меню) - потраченная впустую
работа: тот же компонент придётся стилизовать дважды. Если к моменту начала этого шага
Step 01-04 ещё не все в done/ - остановиться и сообщить, не начинать полировку частично
готового UI.

## Обновление (2026-07-11, после ответа Product Owner) - согласование со Sprint-25-Step-02

Product Owner решил форму picker-а 4 режимов (Sprint-25-Step-02, ранее эскалированная часть):
**квадратные иконки-кнопки, подпись при наведении (hover tooltip)**, не вкладки. Step 02
реализует именно эту СТРУКТУРУ (квадратные кнопки + tooltip), но продолжает использовать
текущий emoji-глиф (🟡🟢🔴🔵, MODE_META.emoji) как содержимое кнопки - оставляет фактическую
замену на настоящий icon-set этому шагу (Step 05). Это единый визуальный язык, разделённый на
два шага по типу работы (структура/раскладка - Step 02; содержимое иконки/общий стиль -
Step 05), не два независимых решения - при исполнении этого шага открыть и перечитать
Sprint-25-Step-02.md (или его ARP/реализацию, если уже в done/) перед тем, как выбирать
конкретные иконки, чтобы форма (квадратная кнопка, hover-tooltip) не разъехалась между двумя
шагами.

## Известная находка, которую нужно явно отметить, не решать тихо

Приложение сегодня использует сырые emoji (🟡🟢🔴🔵) как иконки режимов помощника - см.
MODE_META в apps/studio/src/components/AssistantPanel.tsx (поле emoji, используется в
нескольких местах: сетка/picker режимов - см. также Sprint-25-Step-02 выше, ReaderPanel и т.д.).
Замена на настоящий icon-set (например, lucide-react - самая распространённая в связке с
Tailwind/shadcn, которую вероятно и задокументировал сам ui-specialist skill) - это НОВАЯ
ЗАВИСИМОСТЬ. Согласно правилам проекта ("Adding functionality, dependencies... beyond what was
explicitly requested" - CLAUDE.md, раздел Avoid) и общей практике этого проекта явно фиксировать
выбор зависимости (см., например, как Sprint-25-Step-02 фиксирует выбор resizable-panels
библиотеки в своём ARP) - **зафиксировать явно в ARP этого шага, какая именно icon-библиотека
выбрана и почему, не добавлять её тихо.** Если ui-specialist skill уже задокументировал
конкретный выбор (проверить .claude/skills/literary-studio-ui-specialist/SKILL.md) - использовать
его и сослаться на него в ARP, не выбирать заново с нуля.

Forbidden paths:

  apps/studio/src/app/api/**, apps/studio/src/workspace/useWorkspaceController.ts - чисто
  визуальный проход, никакой бизнес-логики не меняется.
- Никакая новая функциональность (кнопки/поля/операции) сверх визуального единообразия - если
  в процессе находится реальный функциональный баг, не относящийся к дизайну, остановиться и
  сообщить об этом отдельно, не чинить его тихо в рамках этого шага (см. прецедент из Sprint 24
  session handoff - AssistantPanel.tsx regression, обработанный именно так).
- Менять СТРУКТУРУ picker-а (квадратные кнопки + hover tooltip, ряд вместо сетки) - уже сделано
  Sprint-25-Step-02, здесь меняется только содержимое иконки внутри неё, не форма/раскладка.

## Rules

- Опираться на уже задокументированный в literary-studio-ui-specialist визуальный словарь
  (Tailwind/zinc/pill/border вокабуляр, который уже используется по всему проекту) - не вводить
  параллельную систему стилизации (никаких styled-components, CSS-in-JS, новой темизации сверх
  того, что уже есть через prefers-color-scheme).
- Заменить emoji-иконки на компоненты выбранной icon-библиотеки последовательно во всех местах,
  где они используются (не только в picker-е режимов - проверить ReaderPanel и любые другие
  вхождения MODE_META.emoji), сохраняя форму, установленную Sprint-25-Step-02 (квадратная
  кнопка, hover-tooltip с подписью) - не переделывать эту структуру заново.
- Не трогать текст/копирайт (переводы уже закрыты Sprint 15 + Sprint-25-Step-01) - только
  визуальные аспекты (иконки, отступы, консистентность цвета/border/pill-стиля).
- Не редизайнить структуру компонентов (расположение блоков, какие элементы где) - это уже
  решено предыдущими шагами Sprint 25; только визуальный слой поверх существующей структуры.

  чисто.
- Живая проверка (literary-studio-live-verify): реальный npm run dev, скриншоты основных
  экранов до/после (Sidebar, AssistantPanel picker, EditorArea requisites) подтверждающие:
  1. Ни одного сырого emoji не осталось там, где раньше использовался как иконка режима.
  2. Picker сохраняет форму (квадратные кнопки, hover-tooltip), установленную Step 02 - иконка
     внутри заменена, структура нет.
  3. Визуальная консистентность (единый стиль pill/border/spacing) по всем перечисленным
     компонентам - не точечная правка одного файла.
- git status --short - только файлы из Allowed paths.
