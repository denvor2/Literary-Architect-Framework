## Что сделано

Закрыт разрыв между ADR-0012 Decision 5 (Accepted — пользователь должен быть явно предупреждён
при недоступности БД/рассинхроне, не тихий fallback) и фактической реализацией: Sprint-24-Step-06
ошибочно исключил визуальные индикаторы из своего Definition of Done (карточка была написана до
того, как это решение попало в ADR). Использован реальный, уже написанный сигнал Sprint-24-Step-07
(`apps/studio/src/storage/workspaceStorage.ts`, Forbidden path — только прочитан, не изменён):
`export type SyncWarning = "db-unavailable" | "recovered-local-wins"` и
`export function getSyncWarning(): SyncWarning | null`.

1. **`apps/studio/src/workspace/useWorkspaceController.ts`** — добавлено новое поле `syncWarning`
   в возвращаемый хуком объект (запрет Step 06 на новые поля снят явным решением Product Owner
   2026-07-11 — не регресс, а осознанное расширение, прямо оговоренное в самой карточке Step 08).
   Реализация:
   - новый `useState<SyncWarning | null>(null)`;
   - в восстанавливающем эффекте (маунт): после `setWorkspace(restored)`/`setIsLoaded(true)`
     добавлен вызов `setSyncWarning(getSyncWarning())`;
   - в сохраняющем эффекте: `saveWorkspace(workspace).catch(() => {}).finally(() =>
     setSyncWarning(getSyncWarning()))` — прочитано после каждой попытки сохранения, независимо
     от исхода;
   - `syncWarning` добавлен последним полем в возвращаемый объект хука.
   Ни одна мутационная функция не тронута, форма остальных полей не изменена.

2. **`apps/studio/src/components/SyncWarningBanner.tsx`** (новый файл) — минимальный
   conditional-render баннер без новых зависимостей (никакого toast/notification):
   - `warning === null` → `null` (ничего не рендерится);
   - `"db-unavailable"` → жёлтая полоса (`bg-amber-100`/`text-amber-800`, тот же паттерн, что
     "medium"-severity бейдж Critic-режима в `AssistantPanel.tsx`) с текстом: **"База данных
     временно недоступна — изменения сохраняются локально."**;
   - `"recovered-local-wins"` → синяя полоса (`bg-blue-50`/`text-blue-800`, тот же паттерн, что
     инфо-блок AI-подсказки книжных полей в `EditorArea.tsx`) с текстом: **"Обнаружено
     расхождение с базой данных — использованы более свежие локальные изменения."**

3. **`apps/studio/src/app/page.tsx`** — деструктурирован `syncWarning` из хука, отрендерен
   `<SyncWarningBanner warning={syncWarning} />` сразу под `<Header />`, перед основной
   flex-раскладкой (виден при любом активном режиме, включая Focus Mode).

```

**Проверка на реальной сессии Product Owner (`localhost:3000`)** — только наблюдение через
`fetch`, без вмешательства в его вкладку, в четырёх контрольных точках на всём протяжении

```

Оговорка по методу: `curl`/`fetch` возвращает статический HTML-каркас страницы (баннер —

процесс на 3000 не падал и не перезапускался на всём протяжении моей работы (что и было прямым
требованием), и (б) статический каркас страницы не содержит захардкоженного баннера — код
`SyncWarningBanner` рендерит `null`, пока `syncWarning` реально не станет ненулевым, то есть при
штатной работе (без реальных сбоев БД, чем сейчас и была вся сессия Product Owner — его тестовая

ложных срабатываний ожидать неоткуда. Прямое наблюдение реального DOM его вкладки через Playwright
не выполнялось умышленно — по прямому требованию карточки не вмешиваться в его сессию.
